# Wireframe

## context
- Capture the target user, problem, and primary flow for this UI work.

## page_inventory
- List each page or screen that must exist in v1.

## route_map
- Map pages to route or navigation structure.

## key_flows
- Describe the main happy path and blocking alternate paths.

## page_layouts
- Sketch each page in low fidelity: header, body, major sections, and important CTAs.

## responsive_notes
- Describe layout changes for mobile and desktop.

## open_questions
- List any unresolved design questions that block high-confidence implementation.
