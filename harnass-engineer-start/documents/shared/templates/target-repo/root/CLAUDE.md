# Repo Route

Route through `harnass-os/documents/routes/router.yaml` first.
`router.yaml` defines per-phase read lists, gates, and rules.

Read before each phase:
1. The phase runbook (harnass-os/Implement.md, harnass-os/Deploy.md,
   harnass-os/Audit.md, harnass-os/Documentation.md)
2. `harnass-os/documents/design/wireframe.md` and `harnass-os/documents/design/design.md`
   before ui implementation
3. `harnass-os/documents/orchestrator/current.yaml` before phase transitions

Keep context small and prefer repo documents over chat state.
