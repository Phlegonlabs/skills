# ADR Template

## status
- Proposed | Accepted | Superseded

## context
- Describe the problem and the forces acting on the decision.

## decision
- State the chosen direction clearly.

## alternatives
- List the main rejected options.

## consequences
- Record tradeoffs, follow-up work, and migration cost.

## links
- Link the ADR to plans, design docs, runtime docs, or release docs.
