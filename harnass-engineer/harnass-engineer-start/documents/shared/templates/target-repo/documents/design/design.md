# Design

## goal
- Describe the intended UX outcome and the success bar for the UI.

## visual_direction
- Define style, tone, spacing, typography, and color direction.

## page_specs
- For each page, describe content hierarchy and key modules.

## component_rules
- Define reusable component behavior and design-system constraints.

## interaction_states
- Cover idle, loading, empty, error, success, and disabled states where relevant.

## responsive_behavior
- Define breakpoint behavior and mobile-specific adjustments.

## validation_notes
- List the browser-validated flows and screenshots or checks required before shipping.

## implementation_handoff
- State what the implementer must preserve from this design.
